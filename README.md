## 기술스택
-----

```
- Frontend: React, Recoil, Tailwind, kakaomap api
- Backend : Spring-boot, JPA, Gradle
- Database: AWS RDS MySQL
- AI : Flask, Pytorch, Google vision api
- API Test : Postman
- Others: Github, Notion, Zoom,  Figma
```

## 나의 개발 내용

### 전체 페이지 개발 담당

- 프로그래스바 변경 시 즉각적인 데이터 업데이트를 통해 가격 기반 추천 가게를 제공하여 프로젝트 목적성과 사용자의 검색 편의성 향상
- Recoil을 활용하여 회원정보, 카테고리 정보, 가게 정보 등의 전역 상태를 효율적으로 관리
- Kakao Map API 연동 및 카카오 길찾기와 로컬 API 통합으로 정확한 길 안내와 주변 정보 제공
- 지도 위에 마커를 표시하고 사용자가 클릭 시 상세 정보를 제공하여 시각적 정보 제공

<br><br><br>

## 트러블 슈팅

### 문제 상황

- 소주 가격에 따른 가게 추천 프로젝트에서 필터링된 데이터를 불러오는 속도가 느리고 UI 반응성이 떨어져 사용자 경험 저하 및 검색 기능의 효율성 저하

### 문제 접근

- 필터 변경 시 즉각적인 결과 업데이트를 위해 프로그래스바 도입
- Recoil의 비동기 셀렉터를 활용하여 필터링된 데이터를 전역 상태로 관리

### 결과

- 실시간 필터 조건에 따른 결과 확인 가능
- 검색 기능 효율성 크게 개선 및 사용자 경험 향상

<br><br><br>

## 기술적 의사결정

### React
- **선택 이유**: 컴포넌트 기반의 라이브러리로, 재사용 가능한 UI 구성 요소를 만들기 쉽고, 상태 관리가 용이하여 높은 유지보수성 제공
- **활용 예시**: 소주 가격별 가게 리스트 및 상세 정보 페이지의 컴포넌트화

### Recoil
- **선택 이유**: 전역 상태 관리가 용이하고, 비동기 데이터 흐름을 간편하게 처리할 수 있는 라이브러리
- **활용 예시**: 회원정보, 카테고리 정보, 가게 정보 등의 전역 상태 관리

### Tailwind CSS
- **선택 이유**: 유틸리티 퍼스트 CSS 프레임워크로, 빠르게 스타일링을 적용할 수 있으며, 클래스 기반으로 스타일 적용하여 코드 가독성과 유지보수성 향상
- **활용 예시**: UI 컴포넌트에 일관된 디자인 시스템 적용

### KakaoMap API
- **선택 이유**: 한국에서 인기 있는 지도 서비스로, 길찾기 및 로컬 정보 제공 기능이 강력하여 정확한 위치 정보와 경로 안내 제공
- **활용 예시**: 카카오 길찾기와 로컬 API 통합으로 가게 위치의 접근성 향상
